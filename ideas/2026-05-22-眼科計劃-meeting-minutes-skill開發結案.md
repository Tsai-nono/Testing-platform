## 一、專案概述

本次開發針對「114-115年學齡前兒童視力檢查輔導及管理計畫」系列會議，建立可重複使用的 `meeting-minutes` skill，涵蓋會議議程製作與逐字稿轉會議記錄兩大工作流程，目標為減少重複性格式作業、提升文件產出品質與一致性。

- 工作流程 A：會議議程製作（含資訊收集、結構規則、.docx 產出）
- 工作流程 B：逐字稿 → 會議記錄（含解析框架、撰寫原則、.docx 產出）
- 自動觸發關鍵詞設定（「會議議程」「逐字稿」「會議記錄」等）

- `level1` / `level2` / `subItem` / `attendeeLine` 等版面元件
- 可直接 `require` 引用，後續產出文件無需重寫

## 三、已確認格式規範（固定基準）

| 項目 | 規格 | 說明 |
|------|------|------|
| 紙張 | A4，邊距 2cm | 1134 DXA 四邊 |
| 中文字型 | 標楷體 | `font.eastAsia` 只，不設 `name` |
| 英文字型 | Times New Roman | `font.name` 只，不設 `eastAsia` |
| 大標題 | 16pt 粗體置中 | `mixedRuns` + TITLE=32 |

| 第一階編號 | 壹、貳、參…粗體 | `level1()` |
| 第二階編號 | 一、二、三…縮排 | `level2()` |
| 出席者對齊 | Tab stop 2160 DXA | `attendeeLine()` |
| 頁首頁尾 | 無 | — |
| 專家名單附件 | 非例行性 | 視需求附上 |

### v1.1　字型架構修正（重大）
**日期：** 2026-05-18
**原因：** 產出文件中文全部變成 Times New Roman

- **問題根源：** `font: { name: EN, eastAsia: ZH }` 同時設定，Word 將整段套用 Times New Roman，`eastAsia` 失效
- **修正方式：** 純中文 TextRun 只設 `font.eastAsia`；純英數只設 `font.name`；混合文字用 `mixedRuns()` 自動拆分
- **驗證方式：** XML grep 確認 `rFonts` 屬性正確分離後通過

---

### v1.2　議程結構規則確認
**日期：** 2026-05-18
**原因：** 報告事項單項不應加編號

- 報告事項單項 → `subItem()`，無編號
- 報告事項多項 → `level2()`，一、二、三…
- 標題破折號由「－」（全形）改為「-」（半形）
